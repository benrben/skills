# Skills

The **Fathom** suite — five skills on the deep-module principle, sharing the `arch-map` spine (see the [top-level README](../README.md)). Engineer cycle: map → understand → design → code, with review gating changes. All project docs live **only on the spine**, in eleven types ([DOC-TYPES.md](./fathom/DOC-TYPES.md)). The **task board** ([BOARD.md](./fathom/BOARD.md)) makes the cycle trackable — one column per skill, one git worktree per task.

- [`map`](./map/SKILL.md) — observe & record what the codebase IS (modules, depth, edges, leaks, coverage, all signals) and capture the recorded truth as docs of every type. The doc registrar (absorbs the old `adr-writer`) and keeper of worktree truth. A writer of the actual plane alongside `code`.
- [`understand`](./understand/SKILL.md) — read-only guided tour of the map, **its docs**, and **the work in flight** on the board, ending with the named next action. The front door; writes nothing. Explains the **understand** column.
- [`design`](./design/SKILL.md) — decide the deep structure, two modes by request: **improve** an existing shallow module (flag + grill a candidate) or design **new** intended structure (seams, interfaces, sequenced steps). Merges the old `deepen` + `plan`. Owns the **plan** column — sequences each step as a board task with its own worktree.
- [`code`](./code/SKILL.md) — execute a chosen target (refactor shallow→deep, build to a planned interface, or write interface tests), following `MINIMALISM.md`, **inside the task's own worktree**. The only skill that edits source. Owns the **in-progress** column.
- [`review`](./review/SKILL.md) — review a diff/PR (often a task's worktree branch) through the map: modules touched, seams crossed, danger-zones touched without tests, interface erosion. Read-only (may record a `risk`/`postmortem` doc, and move the card); the change gate. Owns the **review** column.

> `fathom/` holds the suite's **shared substrate** — the [`arch-map`](./fathom/arch-map/) MCP spine plus the vocabulary/format docs (`LANGUAGE.md`, `DOC-TYPES.md`, `DEEPENING.md`, `MINIMALISM.md`, `INTERFACE-DESIGN.md`, `CONTEXT-FORMAT.md`, `HTML-REPORT.md`, `BOARD.md`) every skill references. It is not a skill, so it isn't listed above. It also holds the **`craft/`** layer — line-level craft generalized from *Clean Code* (naming, functions, errors, tests, structure, comments, smells), referenced by `code`/`review`/`design`.
